## 一、产品概述

### 1.1 产品定位

**课时管家**是一款面向宝妈的一站式儿童培训班管理工具。帮助她们解决"娃多班多"带来的时间安排混乱、费用管理混乱、课时统计混乱、请假记录混乱四大痛点。

**Slogan**: 别让娃的培训班，成为你的负担

### 1.2 目标用户

- 家有多个孩子的宝妈
- 给娃报了多个培训班的家长
- 需要跨娃、跨班管理时间和费用的家庭管理者

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 多个娃、多个班，上课时间记不清 | 规则驱动的自动排课 + 总览仪表盘 |
| 年费、课时费，记在备忘录里对不上 | 按次分摊的费用统计 + 月度报表 |
| 还剩多少课？心里没数 | 实时课时进度 + 剩余课时预警 |
| 请假了没？课顺延到哪天了？ | 一键请假 + 自动顺延 + 请假记录 |

### 1.4 产品形态

- **跨端混合**: Flutter
- **后端**: LeanCloud BaaS（MVP 阶段模拟数据，后续接入真实 SDK）
- **品牌调性**: 温暖柔和（粉色/珊瑚色/暖色调）
- **目标市场**: 中国大陆（iOS + Android）

## 三、功能详述

### 3.1 用户体系

#### 3.1.1 登录

- 手机号 + 短信验证码登录
- 首次登录进入新手引导
- 已登录用户跳过登录页

#### 3.1.2 家庭共享

- 一个家庭一个主账号（宝妈）
- 可添加家庭成员（爸爸），完全共享读写权限
- MVP 阶段仅支持 2 人

### 3.2 娃档案管理

#### 3.2.1 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | 文本 | 是 | 孩子昵称或真名 |
| 年龄 | 数字 | 否 | 整数 |
| 头像 | 图片 | 否 | 默认自动生成彩色圆形头像 |

#### 3.2.2 操作

- 添加娃 → 进入班级列表
- 编辑娃 → 修改姓名、年龄
- 删除娃 → 级联删除关联班级和课次（二次确认）

### 3.3 班级（培训班）管理

#### 3.3.1 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 机构名称 | 文本 | 是 | 培训机构名称 |
| 班级名称 | 文本 | 是 | 如"大班A班" |
| 课程名称 | 文本 | 是 | 如"美术启蒙" |
| 老师姓名 | 文本 | 否 | 班主任/任课老师 |
| 老师电话 | 文本 | 否 | 联系电话 |
| 总课时 | 数字 | 是 | 课时包总数 |
| 总费用 | 数字 | 是 | 一次性付费金额 |
| 排课规则 | 规则对象 | 是 | 见 3.3.2 |
| 开始日期 | 日期 | 是 | 第一节课日期 |
| 结束日期 | 日期 | 否 | 可选，到期自动结束 |

#### 3.3.2 排课规则

| 规则类型 | 说明 | 示例 |
|---------|------|------|
| 周周期 | 每周固定星期几 | 每周一、三、五 |
| 月周期 | 每月第 N 个周X | 每月第一个周六 |
| 自定义间隔 | 每隔 N 天上一次 | 每14天一次 |

#### 3.3.3 班级状态

- **进行中**: 正常排课，可上课/请假
- **已暂停**: 暂停排课，不显示在活跃列表中
- **已结束**: 课时耗尽或手动结束，归档

#### 3.3.4 操作

- 新建班级
- 编辑班级
- 暂停/恢复课程
- 续费/续班（新建一个班级记录）
- 删除班级（二次确认）

#### 3.3.5 列表筛选与卡片

- 班级列表先按娃筛选，再按科目筛选；切换娃时清空科目筛选。
- 班级卡片展示机构、娃、班级名、剩余课时、课时进度、客单价（单课价）和总费用。

### 3.4 课表生成与管理

#### 3.4.1 自动生成

基于排课规则，在 [开始日期, 结束日期] 范围内自动生成课次。

#### 3.4.2 显示范围

默认显示：当前周 + 前后各一周（共 3 周）

#### 3.4.3 冲突检测

同一娃的两个班在同一时间段时，创建时弹窗提醒，但不阻止创建。

#### 3.4.4 临时停课

支持设置停课起止日期，期间暂停排课，结束后自动恢复。

#### 3.4.5 手动微调

可在自动生成基础上增删个别课次。

### 3.5 上课打卡

#### 3.5.1 流程

1. 系统推送上课提醒（App 内通知 + 系统日历事件）
2. 用户点击"确认上课"
3. 自动扣减 1 课时
4. 可选填写课后笔记/评价

#### 3.5.2 补录

- 课后 24 小时内推送补确认提醒
- 支持手动补录过去课次（标记"补录"）

### 3.6 请假管理

#### 3.6.1 流程

1. 用户选择即将上的课次
2. 填写请假原因（选填）
3. 确认请假
4. 课次标记为"请假"，自动顺延到下一周期
5. 无需老师确认

#### 3.6.2 请假记录

- 可追溯历史请假记录
- 显示请假原因和时间

### 3.7 费用统计

#### 3.7.1 按次分摊

```
单次成本 = 总费用 ÷ 总课时
当月消费 = 当月已上课时数 × 单次成本
```

#### 3.7.2 月度报表

- 当月上课次数
- 当月请假次数
- 当月实际消费金额
- 已消耗课时价值
- 剩余课时价值
- 各班级费用明细

### 3.8 报表统计

#### 3.8.1 总览仪表盘（首页）

| 模块 | 内容 |
|------|------|
| 今日课次 | 当天待上的课，无课则显示提示 |
| 未来3天 | 接下来3天的待上课列表 |
| 本月费用 | 已上课时数×单价 = 当月消费 |
| 剩余课时 | 各娃/各班剩余课时概览 |

#### 3.8.2 月度报表

- 培训累计缴费
- 本月实际课次对应的费用消耗
- 已消耗价值与剩余课时价值
- 各班级总费用、单课价、已消耗金额、剩余价值
- 费用计算口径说明

### 3.9 数据导出

- 支持导出班级列表为 CSV
- 支持导出课次记录为 CSV
- 支持导出月度报表为 PDF（预留）

## 九、关键假设与决策

1. 培训班收费模式以"课时包制"为主（一次性付费）
2. 请假不扣课时，自动顺延到下一周期
3. 用户手动确认上课是主要数据来源
4. 家庭共享是"完全共享"模型（宝妈+爸爸均可编辑）
5. 机构不入驻，所有数据由宝妈手动录入
6. MVP 阶段可使用本地 mock，但当前默认运行链路通过 HTTP service 连接 sibling backend
7. 首次使用引导流程：登录 → 添加第一个孩子 → 添加第一个班级 → 进入首页
