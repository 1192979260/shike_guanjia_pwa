像素级复刻设计方案 v1

目标：在兄弟目录创建 Taro 小程序，接入云托管地址，尽量复刻 Flutter 的业务流程、信息层级和暖色贴纸风格。

1. 架构

Taro app structure
pages/
login, onboarding, home tabs, class detail, class form, settings
store/
Zustand stores mirror Flutter providers: auth, child, class, lesson, settings
services/
request client + API adapters; base URL fixed to cloud hosting
models/
TypeScript interfaces, enum string values, camelCase fields
components/
StickerCard, ChildAvatar, ClassCard, SoftChip, ProgressBar
theme/
warm/fresh/classic tokens mapped from Flutter AppTheme

2. 页面复刻

Navigation parity
启动门禁

班级详情
新增/编辑/续费

主题与提醒
首页五 Tab 保持 Flutter 信息结构：
总览显示今日课程、孩子和课程概览；课表显示今日/近期课次；班级页保留孩子优先、课程其次的筛选；统计页保持费用视角；设置页进入家庭共享、主题、提醒和退出登录。

详情页保持操作闭环：
课次签到、取消签到、请假、补录、手动加课、班级暂停/恢复/结束/删除、编辑后刷新班级和课次。

3. 接口和数据

Cloud hosting integration
Base URL: https://shike-backend-269793-9-1252534988.sh.run.tcloudbase.com
Request: Taro.request, content-type application/json, authorization Bearer token
Response: unwrap data, map error.code/error.message to UI toast or inline error
Storage: Taro storage keys mirror Flutter auth_token, auth_phone, family_id, onboarding_done, theme_skin

4. 视觉策略

Organic sticker style
保留 Flutter 的暖色贴纸方向：沙色背景、圆角贴纸卡片、柔和阴影、圆形头像、胶囊按钮、状态 chip、进度条。小程序里用 SCSS token 统一实现，避免每页散写。
